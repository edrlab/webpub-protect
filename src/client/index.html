  1. the document URLs are predictable (/content/doc[1|2|3].html)
  2. mouse hover or keyboard focus on hyperlinks reveals target URLs
  3. the URLs can be fetched from external requests, without any restrictions (e.g. web browser address bar, or curl -v http://domain.org/content/doc[1|2|3].html)
  4. the HTTP referer header is provided to the destination URL during a cross-origin navigation (no iframe sandboxing))
  5. the browser's "view source" feature or a fetch response's body contains human-readable static document markup
  6. the text can be selected
  7. the document text can be copied to clipboard
  8. the images, links, selected text (etc.) can be dragged and dropped onto a consuming target
  9. the context menu can be displayed with mouse right-click. the web inspector / debugger can be opened via context menu ("inspect element")
  10. the original document markup is revealed in the web inspector

"Protected" documents in iframe:

  1. the document URLs are not predictable (/protect/[RANDOM])
  2. mouse hover or keyboard focus on hyperlinks does not reveals target URLs

    Technique (JavaScript, client side)

  3. the URLs cannot be fetched from external requests without the required contextual HTTP Referer header (however, note that this can easily be worked out, e.g.: curl -v -H "Referer: http://domain.org/protect" http://domain.org/protect/...)
  4. the HTTP referer header is not provided to the destination URL during a cross-origin navigation (iframe sandboxing)
  5. the browser's "view source" feature or a fetch response's body contains no human-readable static document markup (the DOM body is generated dynamically by Javascript)
  6. the text cannot be selected
  7. the document text cannot be copied to clipboard
  8. the images, links, selected text (etc.) cannot be dragged and dropped onto a consuming target
  9. the context menu cannot be displayed with mouse right-click, the web inspector / debugger cannot be opened via context menu ("inspect element")
  10. the original document markup can still be revealed in the web inspector (using the top-level web browser menu to open the debugging tools, and searching for the iframe in the DOM)
